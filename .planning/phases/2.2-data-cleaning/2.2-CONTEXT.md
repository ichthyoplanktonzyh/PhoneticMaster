# Phase 2.2 上下文

## 来源

Phase 2.1 已完成独立训练入口。继续进入反馈、结果页和训练历史前，需要先保证底层训练数据可信，否则后续的 diff 展示、错题统计、最近记录和个性化推荐都会放大数据问题。

当前已知数据相关问题：

- `src/data/wordBank.ts` 仍使用 legacy `WordData` 格式，`src/profiles/en.ts` 需要运行时转换。
- `TrainingItem` 的 identity 已定义为 `display + pronunciation`，但尚无校验入口检查重复或缺字段。
- Profile 的 `phonemes`、`keypadLayout`、词库 parse tokens、L1 difficulty hardPhonemes 之间没有自动一致性校验。
- 中文拼音要求 `pronunciation` 为数字声调 canonical form、`pronunciationAlt` 为显示形式，但尚无数据校验保证。
- L1×L2 映射数据对后续推荐和专项训练有价值，但目前只靠 TypeScript 保证结构，不校验引用质量。

## 关键决策

- Data Cleaning 插入到 Feedback & Session Results 之前，作为 Phase 2.2。
- 原 Feedback & Session Results 顺延为 Phase 2.3。
- Phase 2.1 总结中关于 "Phase 2.2 = Feedback" 的描述是路线重排前的历史记录；当前阶段顺序以 `STATE.md` 和 `ROADMAP.md` 为准。
- 本阶段关注数据质量和校验能力，不改用户训练流程。
- 先清理英语 legacy 数据格式，再建立可复用校验规则。
- 数据校验失败应暴露为开发期问题，不影响已发布用户运行路径。

## 约束

- 不引入外部语料库或远程 API。
- 不扩大词库规模；只做格式统一、规范化和一致性检查。
- 不把 L1 映射数据“语言学正确性”伪装成已验证事实，本阶段只校验结构和引用一致性。
- 如果发现需要人工判断的数据问题，记录到 `CONCERNS.md` 或 phase summary，而不是静默删除。
