## 交付物

- `src/components/PhonemeDiffView.tsx`、`src/components/SessionResultView.tsx`：diff 中的 expected / actual 符号可打开详情。
- `src/components/MinimalPairView.tsx`：当前题和复盘中的 target/contrast 标签可打开详情。
- `.planning/codebase/*`、`AGENT.md`、`CHANGELOG.md`：更新 Phase 3.3 架构、结构、数据模型、测试缺口和项目状态。

## 偏离计划

- 未新建独立音素知识库；本阶段只做 profile + L1 map + 词库 + minimal pair 的聚合 read model，避免维护第二套事实来源。
- 未把 `selectedPhoneme: string | null` 重构为完整 `TrainingTopic`，该项仍留给后续 Training Topic 模型演进。

## 遗留问题

- `src/utils/phonemeDetails.ts` 和 `src/components/PhonemeDetailPanel.tsx` 缺少 fixture-based 单元/组件测试。
- 音素解释仍依赖现有 L1 map 的覆盖度；未覆盖的音素只能显示通用 profile 信息和例词。
- Minimal pair 听辨仍使用 Web Speech API，核心 pair 的标准音频仍是后续体验风险。

## 经验

- 音素详情适合作为 Feedback/Coaching read model，而不是 Training Core 的一部分；它可以解释和唤起训练，但不应参与 scoring。
- 在推荐项里拆开“直接练”和“看详情”两个动作，比把整行点击承担两个含义更清晰。
