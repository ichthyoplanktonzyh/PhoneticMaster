# Phase 3.1 完成总结

## 完成日期

2026-07-02

## 交付物

- `src/types.ts`：新增 `MinimalPairSet`、`MinimalPairOption`、`MinimalPairQuestion`、`MinimalPairSession`、`MinimalPairAnswer`、`MinimalPairResult` 等专项听辨模型。
- `src/data/minimalPairBank.ts`：新增 15 组结构化最小对立体材料，其中英语 8 组、中文 7 组；候选项预留 `audioUrl` 供未来标准音频接入。
- `src/utils/minimalPairs.ts`：新增 pair 筛选、题目生成、答案记录、session 完成和结果汇总服务。
- `src/components/MinimalPairView.tsx`：新增 A/B 听辨视图，支持播放目标、选择候选、即时反馈、正确率和错题复盘。
- `src/App.tsx`：新增 `minimal-pair` app mode；复用 Topic、Words、Voice 控件；支持按当前音素/声调主题进入听辨。
- `scripts/validateData.ts`：数据门禁新增 minimal pair 校验，覆盖 L2、target/contrast phoneme、候选项、notation parsing 和 `audioUrl`。
- `README.md`、`README_EN.md`：补充最小对立体听辨能力、数据规模和标准音频预留说明。
- `.planning/codebase/*`：更新结构、架构、DDD、数据模型、测试和风险文档。

## 偏离计划

- 没有直接改写 `src/l1/zh_en.ts` / `src/l1/en_zh.ts` 中的 legacy `minimalPairs?: string[]`；本阶段改用独立结构化数据源 `src/data/minimalPairBank.ts`，避免继续依赖 `"ship/sheep"` 字符串。
- Minimal pair 结果暂未写入 `phonetic-master-sessions` 最近历史；当前先满足本轮结果统计和错题复盘，M4 本地个性化时再统一进度/掌握度聚合。

## 遗留问题

- 当前播放仍依赖 Web Speech API；发音质量尤其是中文声调和英语细微元音对比需要未来接入标准音频。
- `utils/minimalPairs.ts` 和 `MinimalPairView.tsx` 尚无 fixture-based 单元/组件测试。
- `TrainingTopic` 仍由 `selectedPhoneme: string | null` 表示，后续 Phase 3.2/3.3 可演进为明确 topic 类型。

## 经验

- Minimal pair 数据独立成 `minimalPairBank` 后，既可以从手动主题进入，也可以在未来被 L1-aware 推荐或本地掌握度推荐复用。
- `audioUrl` 必须从一开始进入数据模型，否则后续从 Web Speech API 切换到标准音频会牵动 UI 和训练服务。
