# Phase 3.2 完成总结

2026-07-02

## 交付物

- `src/utils/pinyinBuilder.ts`：新增结构化拼音生成工具，支持零声母、`j/q/x + ü` 正字法转换、轻声 `5` 和多音节追加。

- `src/App.tsx`：中文拼写模式下用结构化拼音面板替换原通用拼音 token 键盘；输入框仍支持自由键入。
- `.planning/phases/3.2-chinese-structured-input/3.2-QA.md`：新增自动检查、规则检查和浏览器冒烟清单。

## 验证

npm run lint
npm run build
npx tsx -e "import { buildPinyinSyllable } from './src/utils/pinyinBuilder.ts'; console.log([buildPinyinSyllable('q','v','4'), buildPinyinSyllable('x','ve','2'), buildPinyinSyllable('','ve','4'), buildPinyinSyllable('n','v','3'), buildPinyinSyllable('l','v','4'), buildPinyinSyllable('m','a','0')].join('\n'))"

## 偏离计划

- 没有新增普通/结构化键盘显式切换。当前采用更保守的兜底方式：中文屏幕键盘为结构化面板，输入框仍可自由输入完整拼音。

## 遗留问题

- 声母 → 韵母 → 声调虽然符合中文教学拆解方式，但仍需要真实学习者反馈默认展示是否直观。
- 结构化输入尚未过滤非法组合，例如 `b + ü`；当前依靠用户选择和 `zhJudge` 判定兜底。
- 没有引入 Vitest；`pinyinBuilder.ts` 后续适合补 fixture-based 单元测试。

## 经验

- 面向学习者的输入辅助不应替代熟悉的直接输入路径；保留自由输入可以降低试验成本。
- 拼音显示正字法和内部解析 token 应继续分离：UI 输出 `qu4` / `xue2`，parser 内部归一到 `q + v` / `x + ve`。
