## 偏离计划

- 起初尝试过独立结构化输入面板，但浏览器 QA 后发现它过重且不符合自然输入节奏；最终改为轻量方案：保留分区键盘，把“声母→韵母→声调”的同步提示放在输入框 placeholder。

## 遗留问题

- 声母 → 韵母 → 声调虽然符合中文教学拆解方式，但仍需要真实学习者反馈 placeholder 是否足够清楚。
- 屏幕键盘仍只是 token 追加；多音节之间需要用户输入空格或手动编辑。

## 经验

- 面向学习者的输入辅助不应替代熟悉的直接输入路径；提示越轻，试错成本越低。
- 拼音显示符号和内部解析 token 应继续分离：UI 可以显示 `ü` 和 `5`，parser 内部继续归一到 `v` 和 `0`。
