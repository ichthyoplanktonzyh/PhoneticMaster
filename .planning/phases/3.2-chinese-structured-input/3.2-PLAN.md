# Phase 3.2: Chinese Structured Input

## 目标

将中文拼音拼写模式从单纯逐字符/逐 token 追加，升级为声母 → 韵母 → 声调的结构化输入辅助，同时保留自由输入兜底。

## 任务

- [x] 3.2.1 确认结构化输入的产品边界：教学辅助，不强制替代自由输入
- [x] 3.2.2 新增中文结构化拼音输入组件
- [x] 3.2.3 生成符合词库 canonical 写法的 tone-number 拼音
- [x] 3.2.4 支持零声母、`j/q/x + ü`、轻声等关键规则
- [x] 3.2.5 在中文拼写模式接入结构化输入，英语仍使用通用 PhoneticKeypad
- [x] 3.2.6 更新文档与 QA 清单

## 退出条件

- [x] 中文拼写模式下可通过声母、韵母、声调组合出一个拼音音节
- [x] 用户仍可直接在输入框输入 `ni3 hao3`
- [x] 生成的拼音可通过现有 `zhJudge` 判定
- [x] 至少覆盖零声母、`qu4` / `xue2`、`nv3` / `lv4`、轻声 `5`
- [x] `npm run validate:data` 通过
- [x] `npm run lint` 通过
- [x] `npm run build` 通过

## 产品判断

声母 → 韵母 → 声调是中文教学中常见的拼音拆解方式，适合作为学习者的结构化辅助；但真实输入习惯通常是直接键入完整拼音，所以本阶段不取消自由输入。
