# Phase 2.1 上下文

## 来源

2026-06-22 产品方向讨论确认：PhoneticMaster 的核心路径应先成为一个完整的语音感知训练器。诊断与推荐有价值，但不应阻塞用户开始训练。

## 关键决策

- MVP 使用纯前端架构，可静态部署到公网。
- L1-aware 推荐是智能教练层，不是第一入口。
- 第一阶段先复用原有 IPA 工具的强项：听音、输入、判定、进度、下一题。
- 后端能力进入 M6 可选增强，仅当同步、标准音频、ASR 或社区内容确实需要时再做。

## 约束

- 当前代码已有 Onboarding 必选 L1/L2 的倾向，实施时需要改成 L1 可选。
- SmartRecommend 在无 L1 时应隐藏或降级提示，不能影响训练器主流程。
- 任何本地存储失败都不应导致训练不可用。
