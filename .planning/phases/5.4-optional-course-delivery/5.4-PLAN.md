# Phase 5.4: Optional Course Delivery & Progress

## 目标

交付可选课程入口、课时导航和本地进度，并通过 activity adapter 复用现有训练与结果能力。

## 任务

- [ ] 5.4.1 实现课程目录、模块和课时 read model/query service
- [ ] 5.4.2 定义 TrainingActivity registry 与 spelling/minimal-pair adapters
- [ ] 5.4.3 实现可选课程入口、课时内容和软顺序导航
- [ ] 5.4.4 从现有 TrainingSession/MinimalPairSession 启动课程活动并接收原生结果
- [ ] 5.4.5 实现 CourseProgress repository 和 TrainingEvidence 关联
- [ ] 5.4.6 支持跳课、回看、直接练习和自由训练证据回填
- [ ] 5.4.7 将 Coach 推荐扩展为可推荐 Lesson 或直接 TrainingActivity
- [ ] 5.4.8 添加依赖边界、进度降级和主要交互测试

## 退出条件

- [ ] 不进入课程也能完成所有原有自由训练流程
- [ ] 课程内拼写/minimal pair 使用与自由训练相同的 session、判定和结果逻辑
- [ ] TrainingSession/MinimalPairSession 不要求 courseId 或 lessonId
- [ ] 用户可以跳过任意课时或直接进入训练，不存在硬锁
- [ ] 浏览进度和训练达标证据被明确区分
- [ ] 本地进度存储失败时课程内容与训练仍可使用
- [ ] `npm run lint`、`npm run test` 和 `npm run build` 通过
