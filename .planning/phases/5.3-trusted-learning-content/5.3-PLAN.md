# Phase 5.3: Trusted Learning Content & Audio

## 目标

为课程和自由训练建立来源可信的学习内容与标准音频层，并提供 TTS 安全降级和语言自定义口音变体。

## 任务

- [ ] 5.3.1 定义 LearningContent、ContentSource、AudioVariant 和地区/口音标签
- [ ] 5.3.2 实现 profile-facing 内容查询与统一 audio resolver
- [ ] 5.3.3 接入课程试点、普通训练、minimal pair 和 PhonemeDetail 播放入口
- [ ] 5.3.4 为试点课程添加小规模许可明确的标准音频
- [ ] 5.3.5 提供播放来源、口音选择和失败降级的透明反馈
- [ ] 5.3.6 扩展数据校验，拒绝授权状态不完整的外部生产内容
- [ ] 5.3.7 测试标准音频、TTS fallback、完全不可用和 profile 缺失内容路径

## 退出条件

- [ ] 有标准音频时优先播放，失败时不中断课程或训练
- [ ] 口音选项来自语言内容声明，不含组件级英语分支
- [ ] 试点课程所有外部内容具备完整来源与分发状态
- [ ] 缺少扩展内容的语言仍可正常自由训练
- [ ] `npm run validate:data`、`npm run lint`、`npm run test` 和 `npm run build` 通过

- `src/profiles/*`

- `src/utils/voice.ts`

- `src/utils/phonemeDetails.ts`

- `public/audio/*`（仅许可明确的小规模资源）
