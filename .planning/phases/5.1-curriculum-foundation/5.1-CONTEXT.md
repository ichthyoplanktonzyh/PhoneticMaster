# Phase 5.1 上下文

## 来源

M4 完成后，项目已具备独立训练、专项练习、反馈、结果与本地个性化，但缺少把学习目标、内容、实施和评价连接起来的课程体系。参考泰勒原理审视：实施和形成性评价已有基础，课程目标与系统化内容仍明显不足。

对 iSpeakerReact 的调研进一步表明，它对课程层级、课时顺序、内容—活动衔接、本地录音/回放和练习交互具有较高参考价值；这种参考属于教学结构与产品机制研究，不等于复制页面表达或获得 Oxford 派生资产的再分发权。

## 关键决策

- M5 定位为“可选课程体系与可信学习内容”。
- `Curriculum` 负责组织目标、内容、活动和评价标准；它不是强制用户路径。
- 自由训练、可选课程和智能教练是三个入口，都可以调用同一个 Training Core。
- 依赖方向必须是 Curriculum → Training application interface；Training Core 不依赖 Curriculum。
- 课程活动只引用现有训练能力，不复制拼写、minimal pair、判定或计分逻辑。
- 课程采用软顺序：可以建议先后次序，但不锁课、不限制训练权限。
- 内容合规不是 Phase 5.3 才处理的媒体附加项，而是 Phase 5.1 的基础数据契约和所有后续内容进入生产构建的前置门禁。
- 课程文本和课程词集优先形成 PhoneticMaster 自有资产；外部资源采用“原创内容 + 开放发音数据 + 逐文件验证的开放音频”组合。
- 首选候选包括：CC BY 开放教材的可改写部分、CMUdict 美式发音数据、Wikimedia Commons/Lingua Libre 的许可明确音频；所有候选仍须逐项核验许可证和例外声明。
- 新语言扩展顺延至 M6，避免与课程领域建模同时引入第三套语言变量。

## 约束

- 不选择课程的用户仍可使用全部基础训练能力。
- `TrainingSession` 和 `MinimalPairSession` 不以 course/lesson 标识作为成立条件。
- `PhonemeDetail` 是可被课时引用的 read model，不等同于 Lesson。
- 课程内容来源和媒体授权必须可追溯；授权不明内容不得进入生产包。
- iSpeakerReact/Oxford 的层级名称和一般教学思想可研究；其课程原文、题目表达、词表编排、音频、视频和图片不得直接进入数据或生产包。
- 免费访问、教育用途、注明来源或第三方仓库公开分发，都不能替代允许本项目再分发的明确许可。
- 本阶段先确定模型、课程地图与接口边界，不构建完整课程 UI。

## iSpeakerReact 参考清单

| 参考维度 | 使用方式 |
|----------|----------|
| 课程组织 | 比较 Course/Module/Lesson 层级、目录和课时粒度，转化为本项目领域模型 |
| 教学顺序 | 研究音素说明、例词、辨音、拼写和复习的衔接，自行设计英语 IPA 课程地图 |
| 活动编排 | 将活动形态映射到已有 TrainingActivity 或后续通用练习契约，不复制训练逻辑 |
| 录音体验 | 参考本地录音、原音对照、回放和删除流程，按浏览器本地优先约束重新实现 |
| 评价反馈 | 参考形成性反馈呈现，评价仍读取本项目 TrainingEvidence |
