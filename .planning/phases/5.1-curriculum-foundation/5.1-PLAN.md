# Phase 5.1: Curriculum Foundation

## 目标

建立可选课程体系的教学模型、课程地图和边界，使目标、内容、训练活动与评价标准能够连接，同时保持自由训练完全独立。

## 任务

- [ ] 5.1.1 用泰勒原理盘点现有目标、内容、实施和评价能力与缺口
- [ ] 5.1.2 定义 Curriculum、Course、Module、Lesson、LearningObjective 和 AssessmentCriterion
- [ ] 5.1.3 定义 LearningContent、TrainingActivity reference、TrainingEvidence 和 CourseProgress 的关系
- [ ] 5.1.4 明确课程活动 registry/application interface，使课程引用现有训练能力
- [ ] 5.1.5 设计课程软顺序、跳课、回看、自由练习证据计入课程进度的规则
- [ ] 5.1.6 绘制英语 IPA 课程地图并选择 Phase 5.2 的最小试点范围
- [ ] 5.1.7 定义内容来源、许可证、署名、修改记录和可再分发元数据，并区分代码/内容/媒体许可证
- [ ] 5.1.8 建立生产内容准入规则：外部内容元数据不完整或许可与公开再分发不兼容时校验失败
- [ ] 5.1.9 定义 PhoneticMaster 课程词集选词规则：词频、音素覆盖、minimal-pair 价值、拼写—发音关系、学习阶段和 L1 难点
- [ ] 5.1.10 形成 iSpeakerReact 机制参考矩阵，明确可借鉴的课程组织/活动模式和不可复制的 Oxford 派生资产
- [ ] 5.1.11 更新 DDD、架构、数据模型、结构、约定、测试和风险文档

## 退出条件

- [ ] 课程目标、内容、活动和评价之间存在可追踪关系
- [ ] 课程可以引用拼写和 minimal pair 活动，而不复制其出题、判定和计分逻辑
- [ ] 无 course/lesson 上下文时现有 Training Core 的接口语义保持完整
- [ ] 课程顺序不形成权限锁，学习者可直接进入任意基础训练
- [ ] 英语 IPA 课程地图与首个试点模块范围已评审
- [ ] 外部内容来源与再分发规则进入数据契约
- [ ] 内容准入门禁能区分“可访问”“可引用”和“可打包再分发”，且不接受代码许可证替代内容许可
- [ ] iSpeakerReact 的参考结果记录为机制矩阵，不包含其 Oxford 派生文本、词表和媒体副本
- [ ] `npm run lint`、`npm run test` 和 `npm run build` 通过

- 新增目标语言
