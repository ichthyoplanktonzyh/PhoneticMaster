# 多语言语音训练平台 Roadmap

## 1. 路线图目标

本路线图将 PhoneticMaster 从英语单语工具演进为 L1-aware 多语言语音训练平台。每个里程碑定义明确的目标、任务、交付物和退出条件。

## 2. 当前上下文快照

| 维度 | 状态 |
|------|------|
| 已完成里程碑 | M0 (原始英语工具) → M1 (多语言架构) |
| 当前里程碑 | M1 ✅ 已完成 |
| 下一个里程碑 | M2 — 训练体验增强 |
| 技术栈 | React 19 + TypeScript + Vite 6 + Tailwind 4 |
| 架构模式 | LanguageProfile-driven SPA |

## 3. 技术选择与决策门

| 决策点 | 选项 | 决策 | 理由 |
|--------|------|------|------|
| UI 框架 | React / Vue / Svelte | React | 项目已有 React 代码库 |
| 样式方案 | CSS Modules / Tailwind / styled | Tailwind | 已有 Tailwind 基础设施 |
| TTS 引擎 | Web Speech API / 后端 TTS | Web Speech API | 零后端依赖，v2 再评估 |
| 状态管理 | useState / Zustand / Jotai | useState | 当前规模足够，不需要额外库 |
| 判定方式 | 字符串比较 / 音素级比较 | 音素级 (phonemeJudge) | 支持 nearMatch 检测 |
| L1 数据来源 | 硬编码 / 远程 API / 社区编辑 | 硬编码 | v1 可控，v2+ 考虑远程 |

## 5. M0 — 原始英语 IPA 工具

> 状态：✅ 已完成。项目起点。

- **目标**：英语美式 IPA 拼写训练
- **覆盖需求**：CORE-001 ~ CORE-005
- **主要任务**：
  - M0.1 IPA 解析器 + 音素分词
  - M0.2 英语音标键盘 + 拼写判定
  - M0.3 Web Speech API TTS
  - M0.4 难度分级 + 词汇筛选
- **交付物**：可运行的英语 IPA 训练 SPA
- **退出条件**：听音 → 输入 IPA → 判定 完整流程可用

---

## 6. M1 — 多语言架构 + 汉语 + L1 推荐

> 状态：✅ 已完成。2026-06-22。

- **目标**：将英语单语工具扩展为 LanguageProfile-driven 多语言平台，新增汉语拼音训练和 L1-aware 智能推荐
- **覆盖需求**：ARCH-001 ~ ARCH-004, LANG-001 ~ LANG-003, L1-001 ~ L1-003
- **主要任务**：
  - M1.1 类型系统重构 (TrainingItem, LanguageProfile, JudgeResult)
  - M1.2 汉语 Profile (声母/韵母/声调 + 拼音解析 + 词库)
  - M1.3 L1×L2 难度映射 (zh→en, en→zh, difficultyMap API)
  - M1.4 UI 重构 (PhoneticKeypad, OnboardingView, SmartRecommend, App.tsx)
  - M1.5 编译验证与修复
- **交付物**：
  - Profile 注册表 + 双语言 profile
  - L1-aware 推荐面板
  - Onboarding 引导流程
  - 零错误编译 + 生产构建
- **退出条件**：
  - ✅ `npx tsc --noEmit` 零错误
  - ✅ `npm run build` 成功
  - ✅ 英语/汉语双语言训练流程可用
  - ✅ 新语言接入仅需 profile + 词库，无框架改动
- **建议投入**：已完成

---

## 7. M2 — 训练体验增强

> 状态：🧭 规划中。

- **目标**：提升训练反馈质量和学习效率，让 nearMatch 和最小对立体训练真正可用
- **覆盖需求**：FEED-001 ~ FEED-003, TRAIN-001 ~ TRAIN-003
- **主要任务**：
  - M2.1 nearMatch 可视化反馈：在拼写模式中显示"哪个音素错了"（利用 JudgeResult.diffs）
  - M2.2 最小对立体训练模式：基于 L1L2Difficulty.minimalPairs 生成对比练习
  - M2.3 音素详情面板：点击音素查看 PAM/SLM 原因、L1 等价音、发音提示
  - M2.4 训练模式增强：添加"跟读"提示、显示释义、标记已掌握音素
  - M2.5 汉语拼写模式优化：声母→韵母→声调三步输入流程（替代直接文本输入）
- **交付物**：
  - diff 可视化组件
  - 最小对立体练习页面
  - 音素详情弹窗
  - 三步式拼音输入
- **退出条件**：
  - nearMatch 时用户能看到具体哪个音素有误
  - 可按 minimal pairs 启动对比训练
  - 汉语拼写模式有三步引导
- **建议投入**：3-5 天

## 8. M3 — 新语言接入

> 状态：🧭 远期。

- **目标**：接入第 3、4 种目标语言，验证 Profile 架构的可扩展性
- **覆盖需求**：LANG-004, ARCH-005
- **主要任务**：
  - M3.1 日语 Profile（假名 + 音调核）
  - M3.2 西语 Profile（西班牙语音系）
  - M3.3 zh→ja / en→ja / zh→es / en→es 难度映射
  - M3.4 通用词库导入工具（CSV/TSV → TrainingItem[]）
- **交付物**：
  - 日语/西语 profile + 词库
  - 新的 L1×L2 映射
  - 词库导入脚本
- **退出条件**：
  - 新语言无需修改框架代码即可训练
  - 至少 1 个新 L1×L2 映射可用
- **建议投入**：5-7 天

## 9. M4 — 训练数据与追踪

  - M4.4 学习统计面板
- **交付物**：
  - 训练历史存储层
  - 掌握度计算引擎
  - 统计可视化组件
- **退出条件**：
  - 用户可查看各音素历史正确率
  - 推荐顺序考虑历史表现
- **建议投入**：3-4 天

| # | 风险 | 可能性 | 影响 | 缓解措施 |
