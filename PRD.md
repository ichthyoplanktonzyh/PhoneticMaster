# PhoneticMaster — 产品需求文档 (PRD)

## 1. 产品概述

PhoneticMaster 是一个轻量、纯前端的多语言语音感知训练器。用户选择目标语言后，可以通过听音拼写、看词听音和后续专项听辨练习，建立目标语言的声音范畴。

核心原则：训练器本身必须完整；L1-aware 推荐是可选教练层，不是开始训练的门槛。

## 3. 目标用户

| 用户画像 | 核心诉求 |
|---|---|
| 第二语言学习者 | 通过反复听辨和输入建立 L2 声音范畴 |
| 中文母语英语学习者 | 区分英语中容易同化到中文音系的音，如 /ɪ/ vs /i/、/v/ vs /w/ |
| 英语母语中文学习者 | 练习声调、卷舌音、ü 等普通话难点 |
| 教师与自学者 | 快速选择语言、难度和训练主题，完成可复盘的一轮练习 |

## 4. 产品分层

| 层级 | 目标 | 当前状态 |

## 5. 已有核心功能

| 编号 | 功能 | 说明 |
|---|---|---|
| CORE-001 | 目标语言选择 | L2 必选，L1 可选 |
| CORE-002 | 听音拼写 | 播放词语，用户输入 IPA 或 Pinyin，系统判定 |
| CORE-003 | 看词听音 | 显示词语和标准记号，用户反复听音并翻页 |
| CORE-004 | 难度与题量 | 支持 basic / intermediate / advanced，题量 1-50 |
| CORE-005 | 主题筛选 | 按英语音素或中文拼音单元筛选训练材料 |
| FEED-001 | 音素级反馈 | 判定 correct / nearMatch / incorrect，并展示 diff |
| FEED-002 | 会话结果 | 完成后展示分数、准确率、错题复盘和最近记录 |
| DATA-001 | 本地历史 | 最近训练结果保存在 localStorage，失败时静默降级 |
| L1-001 | L1-aware 推荐 | zh→en、en→zh 难点映射和推荐面板 |
| TRAIN-001 | 最小对立体听辨 | 结构化 A/B 听辨题，支持本轮正确率和错题复盘 |
| TRAIN-002 | 中文结构化输入 | 中文拼写模式提供声母 → 韵母 → 声调输入辅助，同时保留自由输入 |
| PLAT-001 | 静态发布 | 可部署到 GitHub Pages、Vercel、Netlify、Cloudflare Pages |

## 6. 支持语言与数据

| 目标语言 | 标注系统 | 训练单位 | Basic | Intermediate | Advanced | Total |
|---|---|---:|---:|---:|---:|---:|
| English | IPA | 44 | 881 | 1,527 | 1,680 | 4,088 |
| 中文 | Pinyin | 61 | 131 | 99 | 20 | 250 |

英语词库使用美式 IPA 作为 canonical pronunciation，英式 IPA 作为备用展示。中文词库使用 tone-number Pinyin 作为 canonical pronunciation，带调号拼音作为展示。

## 7. 核心用户流程

```text
打开应用 → 选择目标语言 L2（L1 可选）
         → 选择训练模式 / 难度 / 主题 / 题量
         → 完成训练
            ├── 听音拼写：听音 → 输入 → 判定 → diff
            ├── 看词听音：看词 → 听音 → 翻页
            └── 对比听辨：播放目标 → 在最小对立体中选择（M3）
         → 会话结果 → 错题复盘 / 再练一次 / 换题
```

## 8. 非功能需求

| 需求 | 标准 |
|---|---|
| 静态部署 | 无必需后端、数据库、账号或 API Key |
| 数据质量 | `npm run validate:data` 通过 |
| 类型安全 | `npm run lint` 通过 |
| 构建 | `npm run build` 通过 |
| 存储降级 | localStorage 异常不得阻塞训练 |
| TTS 降级 | Web Speech API 缺失或失败时显示非阻塞提示 |
| 新语言扩展 | 主要通过新增 profile + word bank 完成 |

## 9. M3 需求：专项训练与最小对立体

| 编号 | 需求 | 验收标准 |
|---|---|---|
| TRAIN-001 | 最小对立体数据结构 | 已完成：数据不是 `"ship/sheep"` 字符串，能表达候选项、目标音、对比音和未来音频 URL |
| TRAIN-002 | A/B 听辨视图 | 已完成：每题有播放对象和两个或多个候选项，用户选择后获得即时反馈 |
| TRAIN-003 | 专项入口 | 已完成：用户可从当前音素或声调主题进入 minimal pair 训练 |
| FEED-003 | 专项结果 | 已完成：完成本轮后展示正确率和错题 |
| TRAIN-004 | 中文结构化输入 | 已完成：声母 → 韵母 → 声调三步输入原型，保留自由输入兜底 |
| PLAT-002 | TTS 限制标注 | 已完成：最小对立体明确使用 Web Speech API，并为标准音频预留字段 |

## 10. 非目标

- 用户注册 / 登录
- 云端同步
- 语音识别或发音评分
- 标准音频托管服务
- 付费、课程系统或社交功能
- 后端内容管理

## 11. 风险与决策

| 风险 | 当前处理 |
|---|---|
| Web Speech API 发音质量不稳定 | 支持 voice 选择；minimal pair 数据预留标准音频 URL |
| L1/L2 难点数据专业性强 | 作为可选教练层，结构校验先行，语言学审校后续加强 |
| 拼音解析边界复杂 | 保持 canonical tone-number 数据和解析校验 |
| 无自动单元测试 | 当前依赖 type check、build、data validation 和手动 QA；后续引入 Vitest |
